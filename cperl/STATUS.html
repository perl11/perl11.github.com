cperl status

cperl started Feb. 2015 when :const was added, parrot was killed, the trust in the maintenance capabilities of p5p eroded beyond repair, and it became clear that optimizing for fun is better than waiting for someone else to allow it.

A 5.22 based variant with classes ("perl with classes"), types, compiler support ("compilable perl"), or just a company-friendly perl.

Currently it is about 1.5x faster than perl5.22 overall, >2x faster then 5.14 and uses the least amount of memory measured since 5.6, i.e. less than 5.10 and 5.6.2, which were the previous leaders. While perl5.22 uses the most memory yet measured. But not all of the wanted features are merged. The plan is to support most perl5-compatible perl6 features, improve performance and memory usage, better compiler (B::C) support, and provide better security fixes and maintenance than p5p perl5.

Tested and developed on linux and darwin 64bit. 32bit does not work yet!

The first release based on 5.22.1 is scheduled on Sep 14. 2015.

All tests pass, most of CPAN works, 3 fixes in the distroprefs repo for Variable::Magic and CPAN::Meta::Requirements and version are needed.

Issues will be copied over from our internal stash repo to the public github repo soon.

In the stable master branch are the following major features

  • coretypes
  • shaped arrays
  • static loop optims
  • fast arithmetic overflow
  • convert static method to subs
  • Config as XS
  • strict, attributes, DynaLoader, XSLoader as builtin packages in C
  • changed default hash function to FNV1A
  • optimize XS calls to enterxssub op
  • -DI and -Dk
  • security fixes, e.g. HEK supporting taint now, hash keys keep the tainted info
  • fix ops using lexical $_
  • add some unicode ops
  • improved build system (make -s, faster, CC vs LD confusion)
