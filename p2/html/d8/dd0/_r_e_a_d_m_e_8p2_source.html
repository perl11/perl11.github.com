20  p2 is a she. She is a perl5 and possible perl6/nqp backend, based
21  on why the luck stiff's potion. See README why potion is exciting.
22  Homepage: http://perl11.org/p2/
23

32  Its exciting points for perl folks are:
33 
34  * Should parse most of perl5, just some XS and B tricks not.
35  Since the original perl5 testsuite is not usable for other
36  implementations and compilers it needs to be adjusted. The
37  perl6 testsuite is in a better shape.
38 
39  * signatures, types, native threads for perl5. Hopefully macros also.
40 
41  * Using most of Damian Conway's p5i recommendations:
42  i.e. classes are immutable and final by default.
43  See <http://web.archive.org/web/20040207072905/http://www.yetanother.org/damian/Perl5+i/>
44 
45  * libp2 is usable as a fast backend for nqp/perl6, parsing p6
46  natively should also be possible.
47 
48  * Native and fast calling convention, no XS stack args on the heap.
49  XS is replaced by dlcall (dynamic ffi), and/or shared libraries
50  with normal function calls into the vm.
51 
52  * int, str, num are objects, you can optionally type all args and
53  lexical variables. In fact everything is an object as in parrot,
54  even the compiler, the parser, the vm.
55 
56  * Sized arrays are non-autovivifying and initialized with undef,
57  resp. if typed 0, 0.0, "".
58 
59  my $a[3]; print $a[3] => compile-time error: array out of bounds
60  my int $i[3]; print $a[2] => 0
61  my num $n[3]; print $n[2] => 0.0
62  my str $s[3]; print $a[2] => ""
63 
64  * A new pragma "no magic" applies to all visible objects in scope.
65  {
66  no magic;
67  use Config;
68  print $Config{'ccflags'};
69  }
70  => compile-time error: Invalid use of tie with no magic
71 
72  use
73  {
74  no magic;
75  use Config ();
76  print $Config::Config{'ccflags'};
77  }
78  instead.
79 
80  * p6model: via scoped mixins, :multi, roles. Like Moose, just better
81  and much faster. (roughly estimated 800x faster)
82 
83  * smartmatch works because all data are typed objects.
84  A destructuring-bind alike new match operator which matches
85  structures (list, trees, hashes of hashes) against structures.
86 
87  * Just-in-time compilation for x86, x86-64 and ppc done. arm later.
88 
89  * Intermediate bytecode format and VM. Load and dump code. Decent
90  speed and cross-architecture. Heavily based on Lua's VM.
91 
92  * A lightweight generational GC, based on Basile Starynkevitch's
93  work on Qish. <http://starynkevitch.net/Basile/qishintro.html>
94  The compiler will add hooks for objects leaving scope and add
95  DESTROY method calls if they are visible at
96  compile-time. Run-time created DESTROY methods will not be
97  automatically called.
98 
99  * Bootstrapped "id" object model, based on Ian Piumarta's soda
100  languages. This means everything in the language, including
101  object allocation, parser, compiler and interpreter state are
102  part of the object model. (See COPYING for citations.)
103  In fact are all objects and classes closures, which simplifies
104  scoping, and makes p2 a functional vm.
105 
106  * Planned is an extensibe parser to implement macros and scoped
107  syntax blocks (for ffi, p6, and other languages).
108 
109  * Interpreter is thread-safe and reentrant. I hope this will
110  facilitate coroutines, parallel interpreters and sandboxing.
111 
112  * Small. Under 10kloc. Right now we're like 6,000 or
113  something. Install sloccount and run: make sloc.
114 
115  * Reified AST and bytecode structures. This is very important to
116  me. By giving access to the parser and compiler, it allows people
117  to target other platforms, write compilers, debuggers, code analysis
118  tools and even fully bootstrapped VMs. I'm not as concerned about
119  the Potion VM being fully bootstrapped, especially as it is tied
120  into the JIT so closely.
121 
122  * Memory-efficient classes. Stored like C structs. (Although the
123  method lookup table can be used like a hash for storing arbitrary
124  data.)
125 
126  * The JIT is also used to speed up some other bottlenecks. For
127  example, instance variable and method lookup tables are compiled
128  into machine code.
129 
130  However, some warnings:
131 
132  * Strings are immutable (like Lua) and byte arrays are used for I/O
133  buffers. All strings and symbols are utf8 internally.
134 
135  * Limited floating point support, no bigums yet. ints cannot hold
136  pointers and need to be boxed. This should be transparent though.
137 
138  * Limited error handling.
139 
140  * No signal handling, ffi, thread-safe GC, asynchronuous IO yet.
