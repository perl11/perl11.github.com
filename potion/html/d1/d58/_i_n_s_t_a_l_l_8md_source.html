34 ## ~ building on windows ~
35 
36 Potion's win32 binaries are built using MinGW.
37 <http://mingw.org/>
38 
39 It's a bit hard to setup mingw and gmake on Windows.
40 I usually run a shell under Cygwin and add MinGW
41 to my $PATH.
42 
43 Once that's all done,
44 
45  $ make
46 
47 The easiest way to do this, actually, is on Linux or Darwin.
48 On Ubuntu, if you have MinGW installed,
49 
50  $ make clean; make core/syntax.c
51  $ make config CC=i686-w64-mingw32-gcc CROSS=1
52  $ touch core/syntax.c
53  $ make && make dist
54 
55 This will first create a native greg and core/syntax.c,
56 sets CROSS=1 and cross-compiles with the given CC.
57 See tools/mk-release.sh.
58 make test will not work, you need to copy a make dist tarball
59 to the machine and test it there.
60 
61 Building for win64 does not work yet.
62 cygwin64 misses pthread\_barrier\_t.
63 x86_64-w64-mingw32-gcc fails by creating a wrong config.h

79 ## ~ building with a strict C++ compiler ~
80 
81 potion does not support strict C++ compilers.
82 
83 Either add a C dialect to CC in config.inc (i.e. -std=c89),
84 
85  g++ --help=C; clang++ -x C -std=gnu89
